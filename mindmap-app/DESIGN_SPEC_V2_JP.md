## 1. デザイン哲学

- **主要市場**: 日本語メイン、グローバル（英語ファースト）
- **ブランド印象**: Notion × Figma（アクセシブル、クリエイティブ、企業的でない）
- **ビジュアル原則**: 温かい幾何学的デザイン + 明確な階層 + 暗い優雅さ
- **インタラクション**: フレンドリー、触覚的（ホバーフィードバック、マイクロアニメーション）
- **避けるべき**: 冷たい、無機的、「AIが作った感」

## 3. タイポグラフィシステム

### フォントスタック

```
見出し (H1–H3): Raleway 600/700 (sans-serif, 温かみ, 幾何学的)
  フォールバック: Outfit, DM Sans, Inter
  
本文 (16px+): Noto Sans JP 400/500 (アクセシブルなCJK対応)
  フォールバック: -apple-system, BlinkMacSystemFont, "Segoe UI"
  
等幅 / 数字: IBM Plex Mono または JetBrains Mono (テック向け)
  
マイクロラベル: Raleway 500 10px 大文字, 0.24em letter-spacing
```

### サイズ階層

| 役割 | サイズ | ウェイト | 行高 | 例 |
|------|--------|---------|------|------|
| **H1**（ページタイトル） | 40px | 700 | 1.2 | "何について考えたい？" |
| **H2**（セクション） | 24px | 600 | 1.3 | "AIゲージ" |
| **H3**（サブセクション） | 18px | 600 | 1.35 | "自分の考え" |
| **本文**（標準） | 15px | 400 | 1.6 | ノードラベル、説明文 |
| **小**（副次） | 13px | 400 | 1.5 | ヘルプテキスト、ヒント |
| **マイクロ**（ラベル） | 10px | 500 | 1.2 | "テーマ", "あなたの番" |

### 階層構造

```
H1: 40/700 → H2: 24/600 → H3: 18/600 → 本文: 15/400 → 小: 13/400 → マイクロ: 10/500
```

## 4. コンポーネント設計

### ボタン

**プライマリボタン**（CTA、肯定）
- 背景: `#a78bfa`（紫）
- テキスト: `#1a1a1a`（ダーク、コントラスト用）
- パディング: 16px 24px（縦 × 横）
- 角丸: `12px`（優しい、「ピル型」ほど丸くない）
- フォント: 15px, ウェイト 600
- ホバー: `brightness(1.1)` + `translateY(-2px)` + シャドウ
- クリック: `scale(0.98)`
- 無効: `opacity-40`

**セカンダリボタン**（代替アクション）
- 背景: `#2a2a2a`（ダークBG替）
- 境界線: 1px solid `#3a3a3a`
- テキスト: `#f5f5f5`
- ホバー: `bg-[#3a3a3a]`

**テキスト/ターシャリボタン**
- 背景: 透明
- テキスト: `#a78bfa`（紫）
- ホバー: アンダーライン + `#f5f5f5`

**アイコンボタン**（円形）
- 幅/高さ: 42px
- 角丸: `50%`（完全な円）
- 境界線: 1px solid `#3a3a3a`
- 背景: `#2a2a2a`（ダーク背景時）または `#f5f5f5`（ライト背景時）
- ホバー: 境界線 → `#a78bfa` + `translateY(-2px)`

**ピル/バッジボタン**
- 角丸: `999px`
- パディング: 8px 16px
- フォントサイズ: 13px
- 状態: 未選択（グレー）、選択中（紫/ティール）

### カード & パネル

**カード**（柔らかいコンテナ）
- 背景: `#2a2a2a`
- 角丸: `16px`（丸い、厳しくない）
- パディング: 20px

**モバイルシート**（モバイルでのボトムシート）

- 背景: `#1f1f1f`
- ハンドル: グレーのバー（h-1, w-10）、ドラッグ可能な視覚的ヒント

## 5. レイアウト & 間隔

### グリッドシステム

- **基本単位**: 4px
- **間隔スケール**: 4, 8, 12, 16, 20, 24, 32, 40, 48, 64
- **よくある間隔**:
  - 要素間: 16px
  - カード内: 20px
  - セクション区切り: 24px

### レスポンシブブレイクポイント

| ブレイクポイント | 幅 | 用途 |
|-----------------|-----|------|
| **モバイル** | 320px–639px | スマートフォン |
| **タブレット** | 640px–1023px | ランドスケープスマートフォン、小型タブレット |
| **デスクトップ** | 1024px+ | ノートパソコン、モニター |

### レイアウト変化

- **モバイル**: フル幅、積み重ねレイアウト、パネルはボトムシート
- **タブレット** (sm: 640px): 横並び、狭いサイドバー
- **デスクトップ** (lg: 1024px): 標準的なサイドバー + キャンバスレイアウト

## 6. モーション & アニメーション

### キーフレーム（v1から継承、色を適応）

```
floatUp: 0.5s ease-out (translateY -10px → 0, scale 0.985 → 1)
drawEdge: 0.9s ease-out (曲線へのストロークアニメーション)
typing: 1.2s infinite (ドットバウンス「考え中」インジケータ用)
ringPulse: 1.6s infinite (選択ノード上のパルシング環)
adoptFlash: 0.9s ease-out (採用提案の背景フェード)
```

### ホバー & インタラクション

- **ボタンホバー**: `translateY(-2px)` + `shadow` + 色シフト
- **カードホバー**: 微妙な `scale(1.02)` + シャドウ強化
- **入力フォーカス**: リンググロー + テキストカーソル
- **段階表示**: 子要素は60ms間隔で順に現れる（リスト項目など）

## 9. ダークモード対応

- **キャンバス**: `#111111`（最暗）
- **カード/パネル**: `#1f1f1f` ～ `#2a2a2a`（階層的深度）
- **テキスト**: `#f5f5f5` 主要, `#a0a0a0` 副次
- **アクセント**: 明るい色のまま（紫、シアン、琥珀、ティール）で暗背景とのコントラスト確保
- **シャドウ**: 強め（0.6 opacity）で暗背景に対して深度を表現

---

## 10. アクセシビリティ要件

- **コントラスト比**: すべてのテキスト vs 背景 ≥ 4.5:1 (WCAG AA)
- **フォントサイズ**: 本文最小 13px
- **タッチターゲット**: 最小 44px × 44px
- **ラベル**: すべてのインタラクティブ要素に説明ラベル (aria-label等)
- **色 + 記号**: 色だけに頼らない（アイコン、枠線等も使用）
- **フォーカス状態**: 常に表示、高コントラスト
