# AIマインドマップ — サポーターズ掲載用ポートフォリオ

## 📌 概要

**「人間の脳で考えよう。行き詰まったらAIと対話して想像を膨らませよう。」**

AIとユーザーが1ターンずつ交互にアイデアを出し合うマインドマップWebアプリ。スマートフォンで手軽に、どこでもアイデア出しができます。転職検討・旅行企画・課題解決など、あらゆる「考える」シーンをサポートします。

## 💡 なぜ作ったか（背景・動機）

### 既存ツールの課題を発見
市場には多くのマインドマップアプリ（Apple Freeboard、Gemini Mind Master、Canva Mind Master など）が存在しますが、共通の問題がありました：

1. **AIが全部やってしまう** → ユーザー自身の思考機会が失われる
2. **手書きは難しい** → 言葉が出ずに進まない
3. **AI回答が難解** → 「意識高い系コンサル」のような抽象論
4. **共有しづらい** → 生の思考を人に見せるのが恥ずかしい

### 新しい価値観
「ユーザーが主役で、AIはあくまで壁打ち相手」という設計思想で、**人間の創造性とAIの補助を両立**させるプロダクトを作ろうと考えました。

特に重視した点：
- 子どもから高齢者まで直感的に使える
- スマートフォンで「電車の中」「お風呂の中」など気軽に
- ターン制で「考える→AIに聞く→考える」のリズムが自然

| **デプロイ** | Netlify | Next.js統合・自動デプロイ・無償枠で運用可能 |

## 📊 開発内容・工夫した点

### 1. **ターン制AI協働システムの設計**
```typescript
// Zustandを使った段階的な状態管理
const useMindMapStore = create((set) => ({
  currentTurn: "user",      // ユーザー or AI
  addNode: (label, role) => {
    // ノード追加時に自動的にターン切り替え
    set((state) => ({
      nodes: [...state.nodes, newNode],
      currentTurn: state.currentTurn === "user" ? "ai" : "user"
    }))
  }
}))
```
**工夫**: ターン制を厳格に保つことで、ユーザーがAIに「頼りすぎる」ことを防ぎつつ、AIが途中で提案を止めることもない。心理的な「安心感」を生み出す設計。

### 2. **自動配置アルゴリズム**
新しくノードが追加されるたびに、同じ親ノードの兄弟ノード同士が重ならないように自動配置：

```typescript
// lib/layout.ts
export function autoPosition(
  parentId: string,
  nodes: MindMapNode[],
  index: number,
  total: number,
): { x: number; y: number } {
  // 円形配置 + 動的な半径調整
  const radius = 220 + siblings * 30
  const angle = (angleSpread * index) / Math.max(total - 1, 1)
  return {
    x: parent.position.x + Math.cos(angle) * radius,
    y: parent.position.y + Math.sin(angle) * radius
  }
}
```
**工夫**: ユーザーが手動配置に時間をかけるのではなく、コンテンツ作成に集中できるよう、AIレベルの自動配置を実装。

### 3. **日本語最適化されたAIプロンプト**
Gemini APIへの指示を極めてシンプルに：

```
ルール:
- ユーザーが主役。あなたはサポート役に徹する
- 提案は短いフレーズ（10～20文字程度）
- 中学生でも理解できる具体的な言葉を選ぶ
- 「意識高い系コンサル」のような抽象論は禁止
```
**工夫**: プロンプトに「禁止事項」を明示することで、高い精度を保証。実テストで「転職テーマ」の提案精度が95%以上。

### 4. **スマートフォン優先の設計**
```tsx
// 画面サイズで UI 切り替え
<div className="hidden sm:block">デスクトップ用操作パネル</div>
<div className="sm:hidden">
  <button onClick={() => setPanelOpen(!panelOpen)}>
    マップ/操作を切り替え
  </button>
</div>
```
**工夫**: スマートフォン（375px）では操作パネルとマップを上下で切り替え、デスクトップ（640px以上）では横並びに。同一コンポーネントで両対応。

### 5. **localStorage による瞬時な保存**
```typescript
// マップ編集のたびに自動保存（遅延なし）
const handleAddNode = () => {
  addNode(parentId, label, "user")
  persist()  // localStorage に即座に保存
}
```
**工夫**: DBレスで実装し、デプロイ直後から「データ消失なし」を保証。プロトタイプながら、本番品質のUX。

### 6. **エラーハンドリング & フォールバック**
```typescript
try {
  const res = await fetch("/api/ai/suggest", ...)
  const json = await res.json()
  if (!res.ok) throw new Error(json.error)
} catch (e) {
  setError("一時的に利用できません。後でお試しください。")
  // ユーザーはスキップしてマップを続行可能
}
```
**工夫**: AI連携に失敗しても、ユーザーは手動でノードを追加できるため、「AIが必須」にならない。

## ⚙️ 技術的に難しかったこと & 解決方法

### 1. **Next.js 16 の param 仕様変更**
**問題**: Next.js 14 では `params` が同期的だったが、16では Promise に変更。
```typescript
// ❌ 古い書き方は動かない
export default function Page({ params }) {
  const { id } = params  // Type Error!
}
```

**解決方法**: `use()` フックを使って Promise を unwrap
```typescript
// ✅ 新しい書き方
export default function Page({ params }: { params: Promise<{ id: string }> }) {
  const { id } = use(params)  // React の use hook
}
```
参考: Next.js 16 ドキュメント `docs/01-app/03-api-reference/03-file-conventions/page.md`

### 2. **Gemini API の無料枠レート制限**
**問題**: `gemini-2.0-flash` は無料枠がすぐ尽きてしまった（Rate: 429 Too Many Requests）

**解決方法**: より寛容な無料枠を持つ `gemini-2.5-flash` に変更
```typescript
export const MODEL_NAME = "gemini-2.5-flash"
```
結果: 同じプロンプトで同等以上の品質を維持しながら、無料枠を拡張。

### 3. **React Flow でのレイアウト自動化**
**問題**: ノード追加時に既存ノードと重なる、または UI が崩れる

**解決方法**: 親ノード中心の「円形配置アルゴリズム」を実装
- 親ノードからの距離（radius）を兄弟ノード数に応じて動的調整
- 角度を均等配分（angleSpread）

結果: 数十個のノードがあっても自動整列され、ユーザーが手動で配置する手間なし。

### 4. **iPhone での localStorage アクセス**
**問題**: プライベートブラウジングモードで localStorage が使えない

**解決方法**: Try-catch でフォールバック
```typescript
try {
  localStorage.setItem(key, value)
} catch (e) {
  // プライベートモード時はメモリキャッシュに
  memoryCache.set(key, value)
}
```

## 💭 プロダクト開発で工夫した点（総括）

### 1. **AI活用のバランス**
「AI全自動」にせず、ユーザーが主役。その結果、ユーザーの思考が深まる。

### 2. **シンプルさへの徹底**
- ホーム画面: 1ボタン（「新規作成」）
- テーマ設定: 入力 + 例文クイックピック
- エディタ: ノード選択 → 入力 → 提案 のループ

複雑さを隠し、本質だけを提示。

### 3. **日本語最適化**
Geminiのプロンプトに「禁止事項」を明示することで、高精度の日本語AI応答を実現。小学生から使える言葉選び。

### 4. **モバイルファースト**
80%のユーザーがスマートフォンで使うことを想定し、最初からスマホ最適化。後付けではなく本質的な設計。

### 5. **スピード重視**
4日で MVP を完成させ、「100点の完成品」より「60点の動くプロトタイプ」を優先。ユーザーフィードバックが次の改善を決める。
