## 3. Phase 4: 新規追加機能

### 3-1. NF-01: コミュニティ機能（マップQ&A）
Yahoo!知恵袋型。既存マップに関する質問投稿・回答の場として実装。

- ユーザーは自分のマップ（の一部 or 全体）を公開し、「ここで行き詰まっている」という質問を投稿できる
- 他ユーザーが回答（テキスト or ノード提案の形）を投稿できる
- ベストアンサー選定、いいね機能などQ&Aサイトの基本機能
- **要検討**: 匿名投稿の可否、モデレーション方針（通報機能・NGワード検出）、マップの公開範囲（全体/抜粋）
- 依存: Phase 1（Firebase認証・Firestore）が前提

### 3-2. NF-02: 思考レベルの3D表現
- マインドマップのノード配置を、深さ（テーマからの距離）・広がり（派生数）に応じて3D空間にマッピング
- 2D/3D表示をユーザーがトグルで切り替え可能（既存のReact Flowベースの2D表示は維持）
- 技術候補: `three.js` / `react-three-fiber`、またはReact Flowの3D拡張
- **要検討事項（別セッションで詰める）**:
  - 3D空間での軸の意味（例: Z軸=深さ、XY平面=派生の広がり）
  - カテゴリ別の色分けとの併用
  - モバイルでの3D操作性（タッチでの回転・ズーム）
  - パフォーマンス（ノード数が多い場合の描画負荷）

### 3-3. NF-03: レビュー機能の深掘り元ノード追跡
- 既存のAIレビュー機能（F-08/F-12）で提示される「深掘りすると面白そうな点」について、その元となったノード（該当ノードとその親子関係）へジャンプ・ハイライトできるようにする
- レビュー結果のテキストとノードIDを紐付けて返すようAPI（`/api/ai/review`）のレスポンス形式を拡張

### 3-4. NF-04: 回答時間計測とAIサポートアクセス
- ユーザーがノード入力中の経過時間を計測
- 一定時間（閾値は要調整、例: 30秒〜1分）操作がない場合、「行き詰まっていませんか？AIに相談する」という導線を表示
- Phase 2のトークンゲージ制と連動: 行き詰まり検知時は特別に低コスト（またはトークン消費なし）でAIサポートにアクセスできる救済ルートとする案

### 3-5. NF-05: ノードカテゴライズと伸長診断
- AIまたはルールベースでノードをカテゴリ分類（例: 感情/行動/事実/疑問 など、テーマに応じて動的生成も検討）
- カテゴリ別の分布を可視化し、「このカテゴリはまだ手薄」といった診断コメントを提示
- ※ユーザー指示に2回登場した項目（新規機能・アップデート機能の両方）のため、本書では本項目に一本化

### 3-6. NF-06: ユーザー基本情報の追加
- プロフィール項目: 名前、年齢、職業など（Firebase Authenticationのユーザープロフィール + Firestoreカスタムフィールド）
- 年齢情報は Phase 3 の UP-06（年齢別AI回答調整）で利用
- **要検討**: 必須項目 vs 任意項目、未成年ユーザーへの配慮（個人情報保護方針の整理が必要）

## 4. Phase 3: 既存機能アップデート

### 4-1. UP-01: ゲーミフィケーションの本実装

### 4-2. UP-02: AIトークンゲージ（アクセス制御の再設計）
**最重要・要精密設計。** 現行のターン制（ユーザー→AI→ユーザー）のコンセプトは維持しつつ、以下のルールを追加する。

**確定ルール（ユーザー指示より）**:
1. 人間が回答 → AIが回答、という基本順序は変更しない（ターン制の根幹は維持）
2. AIを1回使用するたびに、人間も1回は回答する必要がある（＝現行仕様のまま）
3. AI提案を**複数個すべて採用（一括採用）した場合**は、次にAIへアクセスするために人間が**複数回**ノードを追加しなければならない（＝一括採用はコストが高い）
4. 個別採用・一部採用の場合は、通常通り人間1ターンでAIに再アクセス可能

**設計案（トークンゲージモデル）**:
| 状態 | ゲージ消費/回復 |
|------|----------------|
| ユーザーが手動でノード追加 | ゲージ +1（回復） |
| AI提案リクエスト | ゲージ -1（消費、ゲージ0では不可） |
| AI提案を個別採用 | 追加消費なし |
| AI提案を一括採用（Accept All） | 追加でゲージ -N（Nは採用数に応じ調整、例: 採用数-1） |

→ 一括採用した分、ゲージが大きくマイナスになり、複数回の手動ノード追加で回復するまでAIにアクセスできない。数値は仮置きのため、実装前にバランス調整が必要。

- 実装箇所: `src/store/mindmap-store.ts`（ゲージ状態管理）、`ControlPanel.tsx`（UI表示・ボタンの活性/非活性制御）
- UIにゲージの現在値を可視化（例: プログレスバー）

### 4-3. UP-03: Apple フィットネス風リング機能
- 「今日の思考活動量」を3種類のリング（例: 手動ノード追加数／AI活用数／マップ完成数）で可視化
- UP-01（ゲーミフィケーション）と統合したダッシュボードとして提供
- ホーム画面 or 新設の「アクティビティ」画面に表示

### 4-4. UP-04: AIパーソナリティ機能
- AIの応答トーン・キャラクター性をユーザーが選択可能に（例: 優しい相棒／クールな分析家／元気な応援団 など）
- プロンプトのシステムメッセージ部分をパーソナリティ設定に応じて動的生成
- NF-06（ユーザー基本情報）と組み合わせ、ターンを重ねるごとに調整する「AIパーソナライズ」（既存F-10, F-19）とも統合

### 4-5. UP-05: ノード整理（自動レイアウト整頓）機能
- 散らかったノード配置を自動整列するボタンを追加
- 既存の `src/lib/layout.ts` を拡張し、階層的レイアウトアルゴリズム（例: dagre, ELK.js）を導入
- 手動配置とワンクリック自動整列をユーザーが使い分けられるようにする

### 4-6. UP-06: 年齢別AI応答調整
- NF-06で取得した年齢情報をもとに、AIの語彙・説明の難易度を調整
- 例: 子供向け＝ひらがな多め・比喩多用、大人向け＝現行の「中学生でもわかる」水準を維持
- `src/app/api/ai/*/route.ts` のプロンプトに年齢帯パラメータを追加

### 4-7. UP-07: ノード接続線（エッジ）の視覚改善
- 現行はシンプルな直線エッジ（`stroke: #94a3b8`）。ベジェ曲線・スマートルーティング（重なり回避）などに変更し、ノードが増えても見やすいレイアウトにする
- React Flowの `type="smoothstep"` や カスタムエッジコンポーネントの検討
